About

I started out as a trainee developer and worked my way into full-stack engineering, spending most of my time on the kind of systems a business can't afford to have go down: payment and order processing, RESTful APIs, and the dashboards customers actually use. At Inokrit Solution I designed eight object-oriented Java backend services with Spring Boot and Apache Camel that processed over a thousand payment and order transactions a day and cut data discrepancies by around 30%.

I work mainly in Java and Python on the backend, with React on the front end — I've built real-time payment and claims dashboards wired into Java payment APIs. I follow secure software development (SSDLC) practices, write automated tests with a TDD mindset, and have set up CI/CD pipelines with GitHub Actions and AWS that took deployments from two hours down to thirty minutes.

I've worked in Agile Scrum teams the whole time, collaborating closely with product management and QA through sprint planning, code reviews, and retrospectives. I like writing code that's easy for the next person to read and maintain, and I'm comfortable troubleshooting defects across dev, test, and production. Right now I'm finishing my Master's in Computer Science and looking for my next software engineering role.
